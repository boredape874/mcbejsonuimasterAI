# Diagrammatic Workflows

Use this to keep broad UI work systematic.

## New UI Flow

```text
intent -> target surface -> design family -> major/mid/subtopic -> closest reference -> IR/tools if layout-heavy -> patch -> validate
```

## Server Form Flow

```text
feature type -> design family -> title route needed? -> preserve fallback -> patch server_form.json -> validate
```

## HUD Flow

```text
HUD change type -> vanilla hud reference -> closest pattern -> side-effect check -> payload protocol if dynamic -> patch UI/sender
```

## restricted reference Flow

```text
neutral ID -> one raw restricted JSON -> extract pattern -> rename everything -> replace restricted textures -> document neutral pattern only
```

Do not edit a visual UI before identifying the design family and closest reference.
